# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## 项目概述

`06_mingzhang_pkm`（新明账）是个人复式记账 iOS App 的前期领域文档仓库。当前阶段核心任务是梳理真实 Excel 记账实践，沉淀领域规则与文档，为未来 iOS 设计与开发提供可直接消费的输入。不做提前的产品扩展或技术架构预设。

## 仓库定位与阶段范围

当前只做三类工作：

1. 梳理账月流程、字段语义、分类体系、固定行、摊销、结转、补记与汇总规则
2. 沉淀 `事实 / 推断 / 待确认` 三类结论
3. 产出可供后续 iOS 设计直接使用的领域文档与样例说明

当前明确不做：不提前扩展成大而全产品规划，不预设数据库、API、页面与技术架构，不抢先定义预算、预测、复杂看板、Agent 等复杂能力。

## 与 01_mingzhang 的关系

旧 `01_mingzhang` 只继承产品上层理念，不继承具体功能方案。可继承的仅有：`要理财，先明账`、资产/负债/收入/支出的财务视角，以及"用户操作贴近真实记账习惯，系统内部再整理严格语义"的方向。

以下内容默认不继承：`J0-J12` 路线图、旧 Web 架构、API/数据库设计、AI/Agent 方案，以及任何没有被当前 Excel 实践验证过的功能设想。若某个旧想法无法回答"它在当前 Excel 实践里对应什么"，则只能作为启发，不能进入当前阶段规范。

## 证据优先级与结论标注

讨论任何对象、流程或规则时，证据顺序固定为：

```
Excel / CSV / XLSX 样例文件 > 你的口述说明 > 当前仓库内已确认文档 > 01_mingzhang 的产品理念
```

旧项目只能提供启发，不能压过当前事实。

**所有结论都必须明确标注为：`事实`、`推断`、`待确认`。不要把猜测直接写成需求或产品结论。**

## 工作原则

- **先复刻已存在实践，再抽象，再优化**。凡是在当前 Excel 中没有稳定对应物的能力，默认不进入当前阶段范围。
- 协作时优先编辑已有文档，而非创建新文档。
- 当前阶段只有以下产出算真正推进项目：澄清一个真实字段或账月流程、确认一条稳定规则、界定一个边界条件、记录一个未解问题及所需证据、或形成一份未来 iOS 可消费的领域文档。
- 不做"通用最佳实践"建议，不引入与当前 Excel 实践无关的记账方法论。
- 讨论前确认：你的建议在当前 Excel 实践中有对应物吗？能指出具体样例行或已有文档依据吗？
