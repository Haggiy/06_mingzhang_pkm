# 新明账关键页面低保真结构设计

> 文档分类：Work Plan / 低保真页面结构设计  
> 状态：讨论沉淀中  
> 说明：本文记录关键页面与功能入口的低保真结构结论，不作为 PRD 或 Spec 权威入口。

## 0. 设计边界

`事实`

- MVP 一级入口为：`首页 / 流水 / 资产负债 / 结果视图 / 设置`。
- `流水日记账` 是唯一正式入账真源。
- 首页、资产负债、结果视图都不是独立真源。
- 当前阶段不做高保真视觉、颜色、字体、图标细节、SwiftUI 实现、数据库或 API 设计。

`推断`

- 本文只沉淀页面结构、功能入口、跳转关系和不承接能力。
- 页面结构必须服务当前 Excel 记账实践，不提前扩展成通用财务软件。

### 全局入口规则

`事实`

整体信息架构采用：

```text
统一账月范围 + 统一流水回溯 + 各入口保留自身职责
```

含义：

- 首页、流水、资产负债、统计共用账月范围选择器。
- 所有来源记录统一回到流水列表页查看。
- 首页、流水、资产负债、统计、设置分别保留自身职责，不互相抢主流程。
- 流水是真源记录入口，但不吞掉资产负债和统计的解释职责。
- 资产负债和统计负责解释、下钻和回溯，不形成第二套账。

`事实`

账月范围选择器统一适用于：

- `首页`
- `流水`
- `资产负债`
- `统计`

账月范围选项统一为：

- `当前账月`
- `全部`
- `YTD`
- `自定义范围`
- `账月列表`

`事实`

`设置` 不使用账月范围选择器。

`事实`

底部浮动操作条只在以下入口出现：

- `首页`
- `流水`

`事实`

`资产负债 / 统计 / 设置` 不放全局浮动操作条。

补充规则：

- 首页：轻量 `+`，展开 `记一笔 / 导入账单 / 查找`。
- 流水：底部浮动条，`搜索 / 点击这里记一笔 / 筛选`，中间 `+` 可展开 `导入账单`。
- 资产负债：只在对象详情页放上下文操作，例如 `调整余额 / 记还款 / 补利息`。
- 统计：只在结果详情页放上下文操作，例如 `查看来源 / 去投资明细账`。
- 设置：无浮动操作。

`事实`

所有 `查看来源 / 查看账目 / 相关流水` 都复用 `流水列表页`，不单独设计新的相关流水页面。

进入流水时带预设筛选条件，例如：

- 首页分类 -> 流水列表页，带账月范围 + 分类筛选。
- 资产类别 -> 流水列表页，带账月范围 + 资产类别筛选。
- 负债对象 -> 流水列表页，带账月范围 + 负债对象筛选。
- 统计变化 -> 流水列表页，带账月范围 + 变化来源筛选。

`事实`

从其他页面跳入流水来源列表时：

- 复用流水列表页。
- 顶部保留返回入口，例如 `返回统计` / `返回资产负债`。
- 显示当前筛选标签。
- 用户可以清除筛选。
- 清除筛选后变成普通流水列表。

`事实`

导入术语统一为：

- `账单导入`：只用于支付宝 / 微信账单导入。
- `数据导入`：只用于新明账自己的数据文件导入。
- 不单独使用含义模糊的 `导入`。

## 1. 首页

### 定位

`事实`

首页定位为：

```text
所选账月范围的收支结构概览 + 最近账目回溯 + 快捷操作入口
```

首页不作为流水真源，不作为完整结果分析页，也不作为资产负债总览页。

### 用户要完成的事

`推断`

- 快速确认当前正在查看的账月范围。
- 查看所选账月范围内的收入、支出、结余和收支结构。
- 从收支结构下钻到一级分类、二级明细和相关账目。
- 查看最近真源账目，并进入单条账目详情。
- 发起手动记账、账单导入或流水查找。

### 首页相关对象清单

`事实`

首页相关对象先收口为 9 个：

1. `首页`
2. `账月选择器`
3. `+ 快捷菜单`
4. `分类底部抽屉`
5. `分类详情页`
6. `记录编辑页 - 新增模式`
7. `导入来源选择`
8. `流水查找页`
9. `账目详情 / 编辑页`

### 信息区

`事实`

首页本体包含：

- 顶部账月范围：例如 `2026年4月 v`、`2026 YTD v`、`2026年1月 - 2026年4月 v`、`全部 v`
- 收支摘要：收入、支出、结余
- 收支结构：收入结构、支出结构
- 最近账目：最近 3-5 条真源记录
- 右下角 `+` 快捷入口
- 底部五个一级入口

`事实`

首页顶部只保留账月范围切换，不放搜索入口，也不放更多入口。

`事实`

首页沿用与流水、资产负债、统计一致的账月范围选择器：

- `当前账月`
- `全部`
- `YTD`
- `自定义范围`
- `账月列表`

`事实`

首页收支分析区只保留 `结构`，不做 `结构 / 趋势 / 明细` 三段切换。

### 收支结构口径

`事实`

首页收支结构只统计收入类和消费 / 费用类支出。

收入结构包括：

- `工作收入`
- `理财收入`
- `其他收入`

支出结构包括：

- `生活必要开支`
- `文娱游购开支`
- `财务费用开支`
- `投资自身开支`
- `其他必要开支`

不进入首页收支结构的对象：

- `资产类支出`
- `负债类增记`
- `负债类减记`
- 投资本金买入和卖出
- 现金储蓄
- 专项储蓄
- 个人账户
- 公积金、社保等资产沉淀部分

### 最近账目口径

`事实`

首页最近账目只展示真源记录：

- `manual`
- `import`
- `investment_feed`

首页最近账目不展示 `engine` 骨架行。

`推断`

- 点击单条最近账目，进入 `账目详情 / 编辑页`。
- 点击 `查看全部`，进入流水入口，并沿用当前首页账月范围。

### 主操作

`事实`

首页右下角 `+` 展开后只包含三项：

- `记一笔`
- `导入账单`
- `查找`

`事实`

`记一笔` 直接进入 `记录编辑页 - 新增模式`：

- 默认账月为当前首页账月范围的终点账月。
- 默认时间为当前时间。
- 不先弹出消费、收入、还款、投资、摊销等类型选择。
- 具体语义由记录表单中的 `收付手段 / 金额 / 收付类型 / 类型明细 / 备注` 表达。

`事实`

`导入账单` 点击后先进入 `导入来源选择`：

- `支付宝`
- `微信`

选定来源后进入对应导入整理流程。导入整理流程在流水入口中继续细化。

`事实`

`查找` 第一版只查流水记录：

- 不查投资明细。
- 不查结果项。
- 不查设置项。
- 查找只定位流水记录，不直接批量修改。

### 次操作

`推断`

- 点击账月范围，打开 `账月范围选择器`。
- 点击收入或支出一级分类，拉起 `分类底部抽屉`。
- 在分类底部抽屉点击 `查看账目`，进入流水并带当前账月范围和分类筛选。
- 在分类底部抽屉点击 `查看全部`，进入 `分类详情页`。
- 在分类详情页点击最近账目，进入对应 `账目详情 / 编辑页`。

### 分类底部抽屉

`事实`

分类底部抽屉只做轻量摘要，不做完整详情页。

抽屉内容包括：

- 一级分类名称
- 金额
- 占比
- 较上月
- 二级明细列表
- `查看账目`
- `查看全部`

`推断`

- `查看账目`：进入流水，并带当前账月范围 + 当前分类筛选。
- `查看全部`：进入分类详情页。

### 分类详情页

`事实`

分类详情页同时适用于收入一级分类和支出一级分类。

- 收入分类使用 `占收入`。
- 支出分类使用 `占支出`。

分类详情页包含：

1. 顶部概览：当前范围金额、占比、较上一可比范围、月均。
2. 近 6 月总额趋势。
3. 近 6 月二级明细构成堆叠图。
4. 当月明细结构：二级明细金额、占比、较上月。
5. 最近账目：当前分类下最近真源记录。

`事实`

分类详情页范围适配规则：

- 单月：展示近 6 月趋势和近 6 月构成。
- YTD / 自定义范围 / 全部：展示范围内按月趋势和按月构成。

`推断`

分类详情页只解释一个分类的变化，不扩展成完整结果视图。

### 跳转关系

```text
首页
├─ 账月范围选择器
├─ + 快捷菜单
│  ├─ 记一笔 -> 记录编辑页 - 新增模式
│  ├─ 导入账单 -> 导入来源选择 -> 支付宝 / 微信导入整理
│  └─ 查找 -> 流水查找页 -> 账目详情 / 编辑页
├─ 点击一级分类 -> 分类底部抽屉
│  ├─ 查看账目 -> 流水列表页（带当前账月范围 + 分类筛选）
│  └─ 查看全部 -> 分类详情页
│     └─ 最近账目 -> 账目详情 / 编辑页
└─ 最近账目
   ├─ 单条记录 -> 账目详情 / 编辑页
   └─ 查看全部 -> 流水列表页（沿用当前账月范围）
```

### 不承接的能力

`事实`

首页不承接：

- 账月阶段。
- 待处理事项列表。
- 上月延续模块。
- 资产负债总览。
- 投资资产摘要。
- 完整结果分析。
- 完整流水管理。
- 预算。
- 预测。
- AI / Agent。
- 设置项维护。
- 资产负债余额编辑。

## 2. 流水

### 定位

`事实`

第二个一级入口用户可见名称为 `流水`。

流水入口定位为：

```text
正式入账记录的查看、搜索、筛选、新增、编辑和导入整理入口
```

`事实`

- 流水日记账是唯一正式入账真源。
- 流水入口不做收入、支出、结余等分析摘要。
- 流水入口不做账月阶段、结账、锁账、核对完成状态。

### 用户要完成的事

`推断`

- 查看当前账月或跨账月范围内的流水记录。
- 通过通用查询和结构化筛选定位记录。
- 手动新增一条记录。
- 查看、编辑或删除允许直接编辑的记录。
- 进入支付宝 / 微信导入整理流程。
- 查看固定行的来源和编辑边界。

### 流水相关对象清单

`事实`

流水入口相关对象收口为 13 个：

1. `流水列表页`
2. `账月范围选择器`
3. `账月列表`
4. `自定义账月范围选择`
5. `筛选面板`
6. `搜索状态 / 搜索结果`
7. `底部浮动操作条`
8. `记录编辑页 - 新增模式`
9. `账目详情 / 编辑页`
10. `导入来源选择`
11. `文件选择 / 导入`
12. `候选记录整理页`
13. `确认导入结果页`

`推断`

`记录编辑页 - 新增模式` 与首页入口共用同一页面定义，不为首页和流水分别设计两套表单。

### 页面结构

`事实`

流水列表页采用：

- 顶部：账月范围作为主标题。
- 主体：记录列表。
- 底部：浮动操作条。

`事实`

流水页顶部不单独显示 `流水` 大标题。

顶部主标题示例：

- `2026年4月 v`
- `2026年1月 - 2026年4月 v`
- `2026 YTD v`
- `全部流水 v`

标题下方可以用小字显示 `流水记录`。

`事实`

流水页上方不做独立工具区。搜索、筛选、添加和导入入口统一由底部浮动操作条承接。

### 账月范围选择器

`事实`

账月范围选择器第一版包含：

- `当前账月`
- `全部`
- `YTD`
- `自定义范围`
- `账月列表`

`事实`

- `账月列表` 用于选择某一个具体账月。
- `自定义范围` 用于选择起止账月。
- 搜索、筛选、排序都作用于当前账月范围。
- 账月范围可以跨月，但每条记录仍保留明确账月归属。

`推断`

账月范围选择器可以采用底部面板，分为：

- 快捷范围：`当前账月 / YTD / 全部 / 自定义范围`
- 账月列表：按年份分组列出具体账月

### 记录列表

`事实`

流水列表不按日期分组。

列表组织方式：

- 单账月：直接展示该账月记录列表。
- 跨账月：按账月分组。
- 跨账月时账月分组行吸顶。
- 不显示日期分组行。
- 不显示当日支出 / 收入小计。

`事实`

记录排序：

- 默认按时间倒序。
- 最近记录在上。
- 可切换为时间正序。
- 排序不改变账月归属和记录时间。

### 记录行

`事实`

记录行采用两行结构：

```text
第一行：时间 + 收付手段 + 金额
第二行：收付类型 / 类型明细 + 备注摘要
```

`事实`

记录行展示字段：

- 时间
- 收付手段
- 金额
- 收付类型
- 类型明细
- 备注摘要

`事实`

- 列表时间默认显示到分钟。
- 详情页完整显示到秒。
- 内部排序和搜索仍按秒处理。

`事实`

金额展示：

- 统一文字颜色。
- 只用 `+ / -` 表达正负。
- 不用颜色推断收入、支出或资产负债语义。
- 金额颜色不承担分类含义。

### 固定行

`事实`

流水列表中的固定行范围为：

- `carry_forward`：跨月继承行。
- `engine`：引擎骨架行。
- `investment_feed`：投资明细账回填流水行。
- 未来其他由专项台账或系统机制生成、但需要进入流水的行。

`事实`

固定行视觉统一为：

- 左侧细竖线。
- 浅底色。
- 不显示原因文字。
- 不按来源分别做不同样式。
- 具体来源和编辑边界进入详情页查看。

`事实`

流水列表不常驻显示来源标记，也不把来源作为表格列。

### 底部浮动操作条

`事实`

流水页底部浮动操作条包含：

- 左侧：搜索。
- 中间：直接 `记一笔`。
- 右侧：筛选。

`事实`

- 点击中间长条直接进入 `记录编辑页 - 新增模式`。
- 中间 `+` 展开更多动作，其中包含 `导入账单`。
- 流水页不在顶部或工具区重复放置新增、导入入口。

`推断`

浮动操作条形态参考：

```text
┌──────────────────────────────┐
│  🔍   + 点击这里记一笔    筛选 │
└──────────────────────────────┘
```

### 搜索与筛选

`事实`

流水查询能力分为两类：

1. `通用查询`：输入文字后直接出结果。
2. `结构化筛选`：用控件选择字段条件。

`事实`

通用查询覆盖：

- 金额
- 收付手段
- 收付类型
- 类型明细
- 备注

`事实`

结构化筛选面板第一版只基于用户可见基础模型，包含 7 项：

- 账月范围
- 时间范围
- 收付手段
- 收付类型
- 类型明细
- 金额范围
- 备注关键词

`事实`

第一版筛选不包含以下机制字段：

- 来源。
- 固定行 / 普通行。
- `record_kind`。
- `record_source`。
- `engine`。
- `investment_feed`。

`事实`

4. 金额

`事实`

- `import`：可直接编辑，可删除。
- `carry_forward`：可编辑，可取消固定。
- `engine`：只读；通过修改来源流水后重算。
- `investment_feed`：只读；修改需进入投资明细账。

`事实`

- `manual / import`：详情页底部可删除，需二次确认。
- `carry_forward`：可 `取消固定`，也可删除本条，需二次确认。
- `investment_feed`：流水中不允许删除，需去投资明细账删除或修改来源交易。
- `engine`：不允许删除，需修改来源记录后由系统重算。

## 3. 资产负债

### 定位

`事实`

资产负债入口定位为：

```text
资产、负债、投资对象的存量查看、范围流量解释、来源回溯和调整入口
```

`事实`

- 资产负债入口不是独立真源。
- 资产、负债余额来自流水和投资明细账结果。
- 资产负债入口可以发起调整、还款、利息补记和投资交易，但最终都必须生成流水记录或投资明细记录。

### 用户要完成的事

`推断`

- 查看所选账月范围终点的资产、负债和净资产。
- 查看所选范围内资产、负债对象的变化。
- 查看资产结构、负债结构和投资专项信息。
- 从资产或负债对象回溯来源流水。
- 对资产或负债发起余额调整，并生成流水调整记录。
- 从负债对象发起还款或补利息 / 费用。
- 进入基金投资明细账维护投资交易和净值记录。

### 资产负债相关对象清单

`事实`

资产负债入口相关对象收口为 12 个：

1. `资产负债首页`
2. `账月范围选择器`
3. `资产类别详情页`
4. `资产余额调整入口`
5. `负债对象详情页`
6. `记还款页`
7. `补利息 / 费用页`
8. `投资专项页`
9. `基金投资明细账`
10. `投资交易编辑页`
11. `净值记录编辑页`
12. `相关流水列表`

### 页面结构

`事实`

资产负债首页第一屏按三块组织：

- `资产`
- `负债`
- `投资`

`事实`

顶部放轻量总览：

- 资产合计
- 负债合计
- 净资产

`推断`

资产负债页顶部可以沿用账月范围作为主标题，不再单独放复杂工具区。

### 账月范围与统计口径

`事实`

资产负债页沿用与流水一致的账月范围选择器：

- `当前账月`
- `全部`
- `YTD`
- `自定义范围`
- `账月列表`

`事实`

资产负债页口径：

- 顶部总览显示所选范围终点的存量。
- 对象明细可展示所选范围内的流量变化。
- 跨月范围下，存量取范围终点。
- `全部` 下，存量取最新账月末。

`推断`

示例：

- `2026年4月`：展示 4 月末存量 + 4 月内变化。
- `YTD`：展示今年截至当前账月末存量 + 年初至今变化。
- `2026年1月 - 2026年4月`：展示 4 月末存量 + 1-4 月变化。
- `全部`：展示最新账月末存量 + 全部历史变化。

### 资产块

`事实`

资产合计必须包含投资资产。

`事实`

资产块展示完整资产结构，包含投资资产这一项。投资明细不在资产块中展开，而是在 `投资` 块中专项承接。

`事实`

资产块按 5 类资产类别列表展示：

- 现金类资产
- 投资资产
- 递延资产
- 个人账户 / 权益沉淀
- 关系型资产 / 应收

每行展示：

- 类别名称
- 期末余额
- 占资产比例
- 范围内净变化

`推断`

点击资产类别行进入资产类别详情页。点击 `投资资产` 行可以进入投资专项页或投资资产详情。

### 资产类别详情页

`事实`

资产类别详情页可以发起 `调整余额`，但不是直接写资产余额。

`事实`

`调整余额` 的本质是生成流水调整记录：

```text
用户填写当前余额
-> 系统计算调整差额
-> 直接生成流水调整记录
-> 资产余额由流水重新计算
```

调整差额：

```text
调整差额 = 用户填写的当前余额 - 系统当前计算余额
```

`事实`

用户只需要填写一个字段：

- `当前余额`

`事实`

调整余额第一版不进入确认页，填完后直接生成流水调整记录。

`事实`

生成后页面落点：

- 仍停留在资产类别详情页。
- 显示反馈：`已生成余额调整记录：+/-xxx`。
- 提供 `查看记录` 入口。
- 点击 `查看记录` 进入新生成的流水记录详情页。

`推断`

系统生成的流水调整记录可默认填充：

- 账月：当前范围终点账月。
- 时间：当前时间。
- 金额：调整差额。
- 收付类型 / 类型明细：根据资产类别预填。
- 备注：余额调整说明。

`事实`

预付费用 / 递延资产第一版不做专项新增入口。

规则：

- 通过通用 `记一笔` 表单录入。
- 使用资产类科目表达，例如 `资产 / 资产类支出 / 长期待摊费用`。
- 后续递延释放通过固定行承接。
- 递延资产详情只负责查看、调整余额和回溯来源。

### 负债块

`事实`

负债块按负债对象列表展示。

示例对象：

- 花呗
- 广发卡
- 招商卡
- 京东白条
- 银行贷款
- 亲朋借款

每行展示：

- 负债对象名称
- 期末剩余负债
- 范围内新增负债
- 范围内已还款
- 范围内利息 / 成本

### 负债对象详情页

`事实`

负债详情页操作：

- 主操作：`记还款`
- 次操作：`补利息 / 费用`
- 次操作：`调整余额`

`事实`

所有操作最终生成流水记录，不直接改负债余额。

`推断`

负债详情页信息区包含：

- 剩余负债。
- 范围内新增负债。
- 范围内已还款。
- 范围内利息 / 成本。
- 相关来源流水入口。

### 记还款页

`事实`

从负债详情页发起 `记还款`。

用户填写：

- 还款金额
- 实际支付手段
- 时间

系统预填：

- 账月：当前范围终点账月。
- 收付类型：`负债类减记`。
- 类型明细：按对象类型预填，例如 `账单还款 / 银行借款还款 / 亲朋借款还款`。
- 备注：`xxx还款`。

确认后生成流水记录。

### 补利息 / 费用页

`事实`

从负债详情页发起 `补利息 / 费用`。

用户填写：

- 金额
- 时间
- 备注

系统预填：

- 账月：当前范围终点账月。
- 收付手段：`账务处理` 或当前负债对象。
- 收付类型：`财务费用开支`。
- 类型明细：`金融利息支出` 或相关费用明细。

确认后生成流水记录。

### 投资块

`事实`

投资块是投资资产的专项展开入口，不代表投资资产被排除在资产合计之外。

`事实`

资产负债顶部 `资产合计` 中的投资资产按成本计量，不按 PV 估值。

口径：

- `资产合计` 使用投资资产成本，保持与流水真源一致。
- `净资产` 也基于投资资产成本计算。
- `投资块` 单独展示 PV 和未实现盈亏，作为估值参考。
- PV 和未实现盈亏不进入流水，也不进入资产合计。

`事实`

投资块指标：

- 投资资产成本
- 投资资产估值 PV
- 已实现收益 / 亏损
- 未实现盈亏
- 本范围买入
- 本范围卖出
- `基金投资明细账` 入口

`事实`

口径：

- 投资资产成本来自投资买卖的入账金额，并进入流水口径。
- 投资资产估值 PV 来自最新净值记录。
- 已实现收益 / 亏损来自卖出计算，并同步流水。
- 未实现盈亏来自 `PV - 持有成本`。
- PV 和未实现盈亏不回填流水。

`事实`

投资块不做：

- 实时行情。
- 非基金投资品类。
- 复杂组合分析。

### 基金投资明细账

`事实`

基金投资明细账第一版记录类型：

- `买入`
- `卖出`
- `净值记录`

`事实`

用户输入字段：

- 日期
- 标的名称
- 交易类别
- 交易金额
- 交易份额
- 单位净值
- 备注

`事实`

系统结果字段第一版展示：

- 入账金额
- 已实现收益或亏损
- 持有份额
- 持有成本

`事实`

`净值记录` 只用于：

- 计算 PV。
- 计算未实现盈亏。
- 展示投资资产估值。

`事实`

PV 和未实现盈亏不回填流水。

`推断`

基金投资明细账页面不突出 `回填` 这个技术词。对用户表达为：

- 维护投资交易。
- 维护净值记录。
- 查看本账月投资结果。
- 系统自动同步到流水。
- 通过低调入口查看相关流水记录。

## 4. 统计

### 定位

`事实`

第四个一级入口用户可见名称为 `统计`。

文档中对应原 `结果视图`，可表达为：

```text
结果视图 / 统计入口
```

统计入口定位为：

```text
从流水和投资明细推导出的结果分析、下钻解释和来源回溯入口
```

`事实`

- 统计入口不是独立真源。
- 统计入口不承接记账维护。
- 统计入口不提供资产或负债余额调整。
- 统计入口不提供投资交易新增或编辑。

### 用户要完成的事

`推断`

- 查看所选账月范围内的收入、支出、结余和资产负债变化。
- 查看收支结构，并下钻到一级分类和类型明细。
- 查看资产负债变化，并回溯来源流水。
- 查看投资结果，并下钻到单个基金标的。
- 从异常结果回到流水或投资明细账修正真源。

### 统计相关对象清单

`事实`

统计入口相关对象收口为 10 个：

1. `统计首页`
2. `账月范围选择器`
3. `结果总览详情`
4. `收支结构详情`
5. `分类详情页`
6. `资产负债变化详情`
7. `变化来源流水列表`
8. `投资结果详情`
9. `标的结果详情页`
10. `投资明细账跳转入口`

### 页面结构

`事实`

统计入口第一屏按四块组织：

- `结果总览`
- `收支结构`
- `资产负债变化`
- `投资结果`

`推断`

统计页顶部沿用账月范围作为主标题，主体展示四块结果分析卡片或分区。

### 账月范围与统计口径

`事实`

统计入口沿用同一套账月范围选择器：

- `当前账月`
- `全部`
- `YTD`
- `自定义范围`
- `账月列表`

`事实`

统计入口口径：

- 单月：展示该账月结果。
- YTD：展示今年至今累计结果。
- 自定义范围：展示范围内累计结果。
- 全部：展示全部历史累计结果。
- 账月列表：跳到某个单月结果。

### 结果总览

`事实`

结果总览展示 6 个指标：

- 收入合计
- 支出合计
- 结余
- 资产净变化
- 负债净变化
- 投资已实现结果

`推断`

结果总览只展示结果性指标，不展开资产或负债对象列表。

### 收支结构

`事实`

统计入口 `收支结构` 复用首页已定的分类详情页模板。

口径：

- 按当前账月范围计算。
- 可用于收入分类和支出分类。
- 点击一级分类进入分类详情页。
- 分类详情页展示该范围下的分类结果和相关账目。

`事实`

分类详情页范围适配规则：

- 单月：展示近 6 月趋势和近 6 月构成。
- YTD / 自定义范围 / 全部：展示范围内按月趋势和按月构成。

### 资产负债变化

`事实`

统计入口 `资产负债变化` 只展示变化分析和来源回溯，不提供调整、还款、补利息等操作。

展示指标：

- 资产增加
- 资产减少
- 资产净变化
- 负债增加
- 负债减少
- 负债净变化

`推断`

点击后进入变化详情，按资产 / 负债类别展示来源流水。

### 投资结果

`事实`

统计入口 `投资结果` 支持按标的下钻。

投资结果总览展示：

- 已实现收益
- 已实现亏损
- 已实现净结果
- 未实现盈亏
- 投资资产成本变化
- 投资估值 PV 变化

`事实`

统计入口增加 `标的结果详情页`。

标的结果详情页用于展示单个基金标的在所选范围内的：

- 历史买入 / 卖出记录
- 净值变化
- BV 变化
- PV 变化
- 持有份额变化
- 已实现收益 / 亏损记录

并提供入口：

- 去投资明细账查看或编辑

`推断`

统计入口的投资结果偏结果解释，不直接新增或编辑投资交易。编辑仍回到基金投资明细账。

### 不承接的能力

`事实`

统计入口不承接：

- 还款或利息补记。
- 投资交易新增或编辑。
- 未实现盈亏回填流水。

## 5. 设置

### 定位

`事实`

设置入口定位为：

```text
账户、科目、数据管理和 App 通用设置的长期配置入口
```

`事实`

设置入口不承接日常记账主流程，不承接资产负债余额编辑，不承接结果修正。

### 用户要完成的事

`推断`

- 维护账户，也就是内部字段中的收付手段。
- 维护科目，也就是资产 / 负债 / 收入 / 支出下的三级科目树。
- 为 L2 / L3 科目维护自然语言语义描述。
- 导出或导入新明账自己的数据文件。
- 清空数据。
- 配置 App 外观、隐私、安全和关于信息。

### 设置相关对象清单

`事实`

设置入口相关对象收口为 14 个：

1. `设置首页`
2. `账户管理页`
3. `账户编辑页`
4. `科目树页`
5. `L2 科目编辑页`
6. `L3 科目编辑页`
7. `数据管理页`
8. `数据导出页`
9. `数据导入页`
10. `数据清空确认页`
11. `外观设置页`
12. `隐私设置页`
13. `安全设置页`
14. `关于页`

### 页面结构

`事实`

设置首页第一屏按三块组织：

- `记账配置`
- `数据管理`
- `App 设置`

`事实`

记账配置主入口简化为两个：

- `账户`
- `科目`

`推断`

设置首页不放记账、导入账单、余额调整等日常操作入口。

### 账户

`事实`

`账户` 对应内部字段中的 `收付手段`。

账户管理按三类分组：

- 资产型账户
- 负债型账户
- 账务处理型账户

支持：

- 新增账户。
- 编辑名称。
- 修改账户类型。
- 停用账户。

`事实`

已被历史记录使用过的账户不直接删除，只能停用。

### 科目

`事实`

`科目` 使用三级科目树管理：

```text
L1：资产 / 负债 / 收入 / 支出
L2：收付类型
L3：类型明细
```

`事实`

L1 固定：

- 资产
- 负债
- 收入
- 支出

`事实`

L2 规则：

- 可新增。
- 可编辑。
- 可停用。
- 可调整所属 L1。
- 可维护自然语言语义描述。

`事实`

L3 规则：

- 可新增。
- 可编辑。
- 可停用。
- 可移动到其他 L2。
- 可维护自然语言语义描述。

`事实`

L2 / L3 的自然语言语义描述用于：

- 解释该科目一般放什么。
- 描述分类边界。
- 为后续 AI / 智能理解提供语义层。

`事实`

这里的语义描述不等同于结构化财务标签，不参与第一版自动规则编排。

`事实`

已被历史记录使用过的 L2 / L3 不直接删除，只能停用。

### 数据管理

`事实`

数据管理第一版包含三项：

- `数据导出`
- `数据导入`
- `数据清空`

`事实`

数据导入与账单导入区分：

- `数据导入`：只导入新明账自己的数据文件，用于迁移、恢复或接续已有数据。
- `账单导入`：导入支付宝 / 微信账单，生成候选流水记录，归属于流水入口。
- 页面文案中不单独使用含义模糊的 `导入`。

`事实`

数据清空是高风险操作：

- 必须强确认。
- 建议先导出。
- 不能做成一键误触。

### App 设置

`事实`

App 设置第一版采用最小集合。

`外观`：

- 主题：跟随系统 / 浅色 / 深色。

`隐私`：

- 隐藏金额。
- 后台隐藏。

`安全`：

- App 锁。

`关于`：

- 版本。
- 数据口径说明。
- 隐私政策。

```text
│  └─ 科目 -> 科目树页

│     └─ L3 科目 -> L3 科目编辑页
├─ 数据管理

   ├─ 外观 -> 外观设置页

   ├─ 安全 -> 安全设置页
   └─ 关于 -> 关于页
```

### 不承接的能力

`事实`

设置入口不承接：

- 日常新增账目。
- 支付宝 / 微信账单导入。
- 资产负债余额编辑。
- 结果修正。

- 预测。
- AI / Agent 自动操作配置。
