# 明账 Excel 实践梳理与重建设计草案

> 文档分类：Research / Excel 历史实践来源  
> 说明：本文保留当前 Excel 工作流分析。其中旧账单补记等内容按 Excel 历史实践理解，不自动等同于 MVP 最新规则。

> 日期：2026-04-16
> 状态：讨论整理版
> 目的：先把当前 Excel 记账实践梳理成清晰的领域文档，作为后续 App 重建的真实起点。

---

## 1. 文档范围

本文档只描述当前真实可运行的 Excel 记账实践，不提前绑定 App 交互，也不提前抽象成通用产品能力。

文档重点回答四个问题：

1. 当前记账实践的核心对象是什么
2. 每个账月是如何开始、处理、结束和跨月继承的
3. 哪些记录来自账单，哪些来自手工补录或自动结转
4. 最终依赖哪些结果视图来判断账是否有价值

---

## 2. 核心原则

### 2.1 账月是强容器

- 每个月对应一张 `流水日记账_YYYYMM`
- 用户不是在连续时间线上记账，而是在某个账月容器里完成当月记账、补录、结转和分析
- 记录的 `日期时间` 是事实字段
- 记录所属的 `账月` 是强约束字段

### 2.2 主表是流水日记账

- `流水日记账_YYYYMM` 是主账本
- 所有资产、负债、收入、支出的账面发生额最终都要落在这张表上
- 月度收支分析、跨月汇总、资产负债统计的统一落点都是这张表

### 2.3 投资采用连续台账

- 投资部分不是直接在月表中维护细节
- 投资细节维护在连续台账 `投资交易记录`
- 月表里的投资相关固定行，由 `投资交易记录` 汇总回填

### 2.4 不使用借贷字段

- 当前实践不显式填写“借 / 贷”
- 复式逻辑由以下三部分共同表达：
  - `收付手段`
  - `当前科目金额正负`
  - `收付类型 / 类型明细`

### 2.5 历史真源头是整理后的 Excel 事实表

- 历史继承时，以每个月整理完成后的 `流水日记账_YYYYMM` 为事实源
- 不要求迁移支付宝 / 微信原始底稿
- 不要求迁移 Python 中间过程数据
- 不要求迁移 Excel 公式布局和样式

## 3. 核心对象

## 3.1 流水日记账_YYYYMM

这是主表。当前最小事实字段为：

- `日期`
- `收付手段`
- `金额`
- `收付类型`
- `类型明细`
- `备注`

其中：

- `日期` 优先记录实际发生时间
- `收付手段` 表示资金来源账户
- `收付类型 / 类型明细` 表示资金去向和经济意义
- `金额` 是当前科目的带符号发生额
- `备注` 承接业务说明、周期信息、人工语义和部分统计辅助信息

## 3.2 投资交易记录

这是连续台账，不按账月切断。典型字段包括：

- `日期`
- `标的名称`
- `交易类别`
- `交易金额`
- `交易份额`
- `入账金额`
- `单位净值`
- `持有成本(T)`
- `投资收益`
- `持有份额`
- `BV`
- `PV`
- `备注`

其中：

- 交易事实是核心
- `BV / PV / 收益` 既是分析结果，也是当前 Excel 中的重要输出
- 主账本中的投资固定行由这张表支撑

## 3.3 时间

当前表中名为 `日期` 的字段，本质上应理解为 `时间` 字段。

它支持到秒，格式通常写成 `yyyy/mm/dd hh:mm:ss`。

规则：

- `时间` 一般代表记录的实际发生时间
- 导入记录如果原始账单提供到秒，应保留到秒
- 如果原始记录只有日期、没有具体时间，则记为当天 `00:00:00`
- 手动新增记录时，默认使用当前时间
- 固定行也尽量改成真实时间；拿不到真实时间时，允许使用约定时间占位
- 如果后续知道了更准确的时间，可以再改；但在当前实践中并不强求绝对一致

说明：

- `账月` 是强约束，决定记录属于哪个月账本
- `时间` 是弱事实字段，优先真实，但允许为了操作便利保留近似时间
- 产品字段名可以继续沿用 `日期`，但领域语义上应按 `时间戳` 理解

## 3.4 收付手段

`收付手段` 代表资金来源账户。当前实践中主要有三类：

### 资产型

- `现金`
- `电子钱包余额`

### 负债型

- `花呗`
- `广发卡`
- `招商卡`
- `平安卡`
- `北京卡`
- `蚂蚁卡`
- `京东白条`

### 账务处理型

- `账务处理`

说明：

- 这些值来源于对支付宝 / 微信账单中“付款方式”的清洗和聚合
- `账务处理` 不是实际资金账户，而是用于没有实际资金流动、但需要进行账务确认、结转、摊销、归集或汇总时使用
- `非固定行` 必须有 `收付手段`
- `固定行` 在尚未实际发生前允许 `收付手段` 为空
- 固定行一旦实际发生，就需要补上真实账户，或补上 `账务处理`

## 3.5 金额

`金额` 是当前科目的带符号发生额。

规则：

- 正数表示该科目的正常发生额
- 负数表示该科目的反向发生、冲销、退款、回退或释放
- 正负号不直接等于收入或支出
- `收付手段` 不决定金额正负，只说明这笔发生额由哪个资金账户或账务处理方式承接
- 资产、负债、收入、支出的影响方向，由 `收付手段 + 收付类型 / 类型明细 + 金额正负` 共同推导

例如，信用卡消费：

```text
2025/11/3 08:41    招商卡    35.00    生活必要开支    交通通信费    公交地铁
```

表示：

- `交通通信费` 正常发生 35
- 这笔消费由 `招商卡` 承担
- 它不是现金流出
- 后续是否形成负债类增记，要看次月是否出现 `账单补记` 行

例如：

```text
2026/1/1 00:00    账务处理    -4587    资产类支出    长期待摊费用    预付房租-博学西路6号院6/3/201-计提本月房租服务费开销
```

表示：

- `长期待摊费用` 这项资产类科目发生反向变动
- 对业务含义而言，它表示 `预付房租` 这项资产减少了 4587

再例如：

```text
2026/1/15 00:00    账务处理    -283.1916667    资产类支出    其他资产类支出    友望洗地机 1/6 (202509-202602)
2026/1/15 00:00    账务处理    283.1916667    文娱游购开支    大件购置费    友望洗地机 1/6 (202509-202602)
```

表示：

- 待摊资产减少
- 消费开支增加
- 两条合起来表达“待摊资产转为消费”

## 3.6 收付类型与类型明细

`收付类型` 是一级经济科目，`类型明细` 是二级经济科目。

当前实践中的一级分类分为三大域：

- 消费类
  - `生活必要开支`
  - `文娱游购开支`
  - `财务费用开支`
  - `投资自身开支`
  - `其他必要开支`
- 收入类
  - `工作收入`
  - `理财收入`
  - `其他收入`
- 资产负债类
  - `资产类支出`
  - `负债类增记`
  - `负债类减记`

例如：

- `生活必要开支 -> 伙食费 / 房租费 / 交通通信费`
- `工作收入 -> 到手工资 / 奖金 / 社保 / 公积金`
- `资产类支出 -> 现金储蓄 / 专项储蓄 / 金融资产投资 / 长期待摊费用`
- `负债类增记 -> 账单补记 / 账单分期 / 银行借款`
- `负债类减记 -> 账单还款 / 银行借款还款 / 亲朋借款还款`

规则：

- `类型明细` 必须归属于一个 `收付类型`
- `类型明细` 不是标签，而是有唯一归属的二级科目
- 同一个 `类型明细` 不跨 `收付类型` 复用

## 3.7 备注

`备注` 不是纯展示字段，而是允许承载重要但不结构化的信息。

当前实践中，备注通常承接：

- 业务说明
- 周期范围
- 分期信息
- 资产或还款说明
- 人工判断语义
- 某些统计辅助信息

规则：

- 对稳定对象，通常会采用稳定写法，便于筛选和公式计算
- 例如：
  - `公积金*`
  - `预付房租*`
- 这类稳定备注目前会被用于 `SUMIFS`、关键字匹配和正则统计
- 先承认这是当前 Excel 实践中的必要权衡，不额外抽象为新字段

## 4. 复式表达方式

当前实践中的复式不是标准会计分录录入方式，而是“规则驱动的单行复式模型”。

一条记录的经济意义由以下四层共同表达：

1. `收付手段`：资金来自哪个账户
2. `金额`：当前科目的带符号发生额
3. `收付类型`：一级经济科目
4. `类型明细`：二级经济科目

例如：

```text
2025/9/5 11:54    电子钱包余额    20.00    生活必要开支    伙食费    食堂-午餐
```

含义：

- 钱从 `电子钱包余额` 出去
- 去向是 `生活必要开支 / 伙食费`
- `伙食费` 正常发生 20
- 现金资产减少是由 `收付手段 = 电子钱包余额` 和开支科目共同推导出来的，不是由金额负号表达的

再例如信用卡消费：

```text
2025/11/3 8:41    招商卡    35.00    生活必要开支    交通通信费    公交地铁
```

含义：

- 这笔消费由 `招商卡` 这个负债账户承担
- 去向是 `生活必要开支 / 交通通信费`
- 若金额为负数，则表达退款
- 消费发生时不直接生成负债类增记账目行；负债正式入账依赖后续 `账单补记`

再例如摊销：

```text
2026/1/15 00:00    账务处理    -283.1916667    资产类支出    其他资产类支出    友望洗地机 1/6 (202509-202602)
2026/1/15 00:00    账务处理    283.1916667    文娱游购开支    大件购置费    友望洗地机 1/6 (202509-202602)
```

含义：

- 第一条表示待摊资产减少
- 第二条表示消费开支增加
- 两条合起来表达“待摊资产转为消费”

## 5. 账月的真实工作流

## 5.1 起新账月

新账月的开始不是创建空白表，而是：

1. 复制上个月已经处理好的 `流水日记账`
2. 修改 sheet 名称为新账月
3. 删除非固定行
4. 保留跨月固定行和仍在有效期内的条件固定行

在 Excel 中：

- 固定行通常位于某个高亮行的上方
- 有时间范围的固定行会在备注中明确写出周期
- 到期后手动删除

## 5.2 处理账单

账单处理是核心入口，流程为：

1. 导出支付宝 / 微信账单
2. 对账单进行处理
3. 主要工作是分类、识别和手动补必要的复式分录
4. 形成处理好的记录后，粘贴回对应账月的 `流水日记账`

## 5.3 补录账单外交易

不在支付宝 / 微信账单中的交易，必须手动补录。主要包括：

- 工资
- 五险一金
- 信用卡和贷款还款
- 投资记录
- 银行卡或其他渠道支出
- 结转和调整类记录

## 5.4 月末准备下个月

月末不是锁账，而是：

1. 完成当月记录整理
2. 保留仍需跨月存在的固定行
3. 复制当前表，作为下个月模板起点

当前实践中没有“关账 / 反结账”的强机制，历史月可随时修改。

## 6. 固定行与跨月继承

所谓“月度模板”，本质上不是独立模板系统，而是：

`跨月继承的固定行集合`

在当前 Excel 实践里，固定行只按是否长期存在和是否有明确期限来理解，不额外发明复杂类型字段。

## 6.1 固定行的最小定义

固定行就是：

- 会复制到下个月的一行
- 它的金额要么直接写死，要么每月确认后填写，要么由公式自动重算
- 有些会一直存在，有些会在到期后删除

当前实践中，固定行分为两大类：

1. `永久固定行`
2. `周期固定行`

## 6.2 永久固定行

永久固定行的含义是：

- 为了复式记账和核算闭环，整个系统必须长期存在
- 一旦定下来，原则上不是为了某个短期业务临时创建的
- 它们更像核算骨架，而不是提醒或便捷录入项

典型内容包括：

- 结转类固定行
- 确认负债类固定行
- 确认资产类固定行
- 现金流量变动骨架
- 投资相关汇总骨架

典型例子：

```text
2026/1/1 0:00    账务处理    0.00    负债类增记    账单补记    上月广发卡账单补记
2026/1/31 0:00   电子钱包余额    0.00    资产类支出    金融资产投资    本月金融资产投资-买入
2026/1/31 0:00   账务处理    0.00    资产类支出    现金储蓄    现金流量变动-负债类增记
```

说明：

- 这些行会一直跨月继承
- 对于能用公式算出的值，当前 Excel 实践中直接依赖公式自动重算，不手工维护金额

## 6.3 周期固定行

周期固定行的含义是：

- 为了记账方便，不需要每个周期反复重写
- 它们不是核算骨架，但会周期性出现
- 有些会长期持续，有些有明确截止时间

周期固定行分两种。

### 6.3.1 周期固定行-无固定期限

这类行会一直带到下个月，直到用户主动取消。

典型内容包括：

- 工资确认
- 负债还款提醒
- 房租摊销
- 部分长期稳定存在的周期项目

说明：

- 这类行可能金额固定，也可能每月确认
- 某些记录在实际执行前允许 `收付手段` 为空
- 例如还款行在未实际支付前，只承担提醒作用

关于房租，当前 Excel 的具体做法是：

- 房租本身属于一项周期固定行
- 但为了按日统计消费趋势时不被单日大额房租扭曲，实际会在月内拆成多条记录均匀落入若干日期
- 这里保留的是 Excel 里的真实实践，不额外上升为新的规则字段

### 6.3.2 周期固定行-有固定期限

这类行有明确开始和结束时间，到期后删除。

典型内容包括：

- 分期付款
- 长期待摊
- 有明确到期月的周期项目

典型例子：

```text
2026/1/15 0:00    账务处理    (261.94)    资产类支出    其他资产类支出    宜米饮水机 1/6 (202509-202602)
2026/1/15 0:00    账务处理    261.94    文娱游购开支    大件购置费    宜米饮水机 1/6 (202509-202602)
```

这类固定行的特征：

- 会跨数个月继承
- 到期后手动删除
- 通常在备注中写明周期范围

## 6.4 不属于固定行的内容

以下内容不应再混入固定行体系。

### 6.4.1 临时项

临时项不属于通用固定行。

例如：

- `大庚应负担生活费用-临时，应明细统计应负担情况`
- `大庚待业期间额外承担共同-生活-开支`

这类记录更适合当月手工补录或作为专项临时项处理，而不是进入通用固定行规则。

### 6.4.2 真实账单流水

重复发生的真实消费流水也不属于固定行。

例如：

- 订阅类账单流水
- 可从支付宝 / 微信 / 信用卡账单中直接导出的真实消费

当前 Excel 中曾把这类记录先放在固定行区域，主要是为了在月初先看到固定开支大概有多少，后续再手动删除。

但从实践口径上看，这类记录更应该随着账单导入进来，而不是继续作为固定行继承。

### 6.4.3 临时口径修正

`父母亲人赡养费预支` 不视为临时项，而归入 `周期固定行-无固定期限`。

## 7. 负债处理实践

## 7.1 刷信用卡消费

消费发生时，直接记主账本。

示例：

```text
2025/11/3 8:41    招商卡    35.00    生活必要开支    交通通信费    公交地铁
```

理解方式：

- `收付手段` 记录资金来源
- `收付类型 / 类型明细` 记录资金去向
- `金额` 记录对账户的影响
- 若金额为负数，则表示消费退款

## 7.2 次月账单补记

信用卡消费会在次月形成一条固定行，用来汇总上月该卡消费总额。

示例：

```text
2025/12/1 0:00    账务处理    17,290.97    负债类增记    账单补记    上月招商卡账单补记
```

这条记录通常通过 `sumifs` 统计上月该卡的消费发生额生成。

## 7.3 还款记录

还款记录在当前实践中是“待执行固定行”。

示例：

```text
2025/12/20 0:00        392.86    负债类减记    账单还款    花呗本期账单还款
2025/12/23 0:00        3,488.69  负债类减记    银行借款还款    北京银行贷款还款(36) 202409 -> 202708
```

关键特征：

- 这类记录的 `收付手段` 可以为空
- 含义不是数据错误，而是“还款义务已确认，但真实支付尚未发生”
- 实际支付时，在原记录上补上 `收付手段`
- 只有补上真实资金来源后，资产账户统计才会受到影响

## 7.4 利息和分期

- 按时还款通常不产生手续费和利息
- 若办理分期或贷款，会产生专门的负债或财务费用处理项

示例：

```text
2025/12/23 0:00    账务处理    155.36    财务费用开支    金融利息支出    北京银行贷款利息(36) 202409 -> 202708
```

## 9. 投资与长期待摊实践

## 9.1 长期待摊

示例：

```text
广发卡    1200    资产类支出    长期待摊费用    12个月健身房费用(202601-202612)
```

按月结转时：

```text
账务处理    -120    资产类支出    长期待摊费用    12个月健身房费用(202601-202612)
账务处理    120     投资自身开支    健身训练费    本月健身卡费用
```

这说明当前实践支持：

- 先资产化
- 再按月摊销

## 9.2 投资主从关系

投资部分有明确的主从关系：

- `流水日记账` 是主表，记录一切资产负债和收入支出的账面发生额
- `投资交易记录` 是明细台账，用来支撑资产分析
- 在投资记账中，手动维护的是 `投资交易记录`
- `流水日记账` 中的投资相关金额由公式和固定行自动汇总生成

典型固定行包括：

```text
本月金融资产投资-买入
本月金融资产投资-卖出
本月金融资产投资-已计入专项储蓄部分
本月金融资产投资-赎回亏损-权益类亏损
本月金融资产投资-赎回亏损-债券类亏损
本月金融资产投资-投资收益-权益类收益
本月金融资产投资-投资收益-债券类收益
```

这些记录用于支撑账面价值计算。

## 10. 主账本与投资台账的关系

当前实践中两张表的职责划分明确：

## 10.1 流水日记账的职责

- 记录所有资产、负债、收入、支出的账面发生额
- 作为月度收支分析和跨月汇总的统一事实落点
- 承担结转、还款、工资、长期待摊、投资汇总等账面结果

## 10.2 投资交易记录的职责

- 维护投资买入、卖出、份额、净值等连续事实
- 支撑持仓明细、BV、PV、收益等投资分析
- 为主账本中的投资固定行提供汇总来源

结论：

- 投资交易记录不是第二本总账
- 它是投资领域的连续明细台账
- 主账本中的投资结果由它回填

## 11. 核对与纠错实践

当前实践不追求复杂对账机制，核心是 `账实核对后修正`。

主要核对对象：

- 投资持仓
- 大件资产账户
- 公积金账户
- 社保账户
- 电子钱包余额
- 负债余额

发现不一致时，通常采用以下方式修正：

- 改原记录
- 补一条手工记录
- 加一条调整行
- 若问题属于投资明细，则修改 `投资交易记录`

当前实践对高频流水余额并不追求过度精细核查，更重视：

- 投资持仓是否正确
- 大件资产和负债余额是否正确

## 12. 结果视图

当前 Excel 实践最终依赖以下结果视图。

## 12.1 收支

### 月度收支分析

- 按科目统计当月发生额
- 用于回答“这个月花了多少，花在哪些科目上”

### 跨月度分析

- Rolling Table
- 列为月份，行为科目
- 统计发生额
- 用于观察更大周期的收支变化

## 12.2 资产

### 持仓明细

- 按类别分组
- 统计 `BV / PV / 收益`

### 跨月资产趋势

- 用于观察资产变动趋势

## 12.3 负债

### 负债余额汇总

- 统计当前余额
- 统计本月还款情况
- 统计利息和利率

## 13. 对 App 重建最重要的约束

虽然本文档不直接定义 App 行为，但它已经给出几条不能违反的约束：

1. `账月` 必须是强工作上下文
2. `流水日记账_YYYYMM` 必须是主账本
3. `投资交易记录` 必须是连续台账，不按月切断
4. 投资相关月度结果必须由投资台账回填主账本
5. 还款、工资、结转、摊销等账单外交易必须被当成一等公民对象
6. 固定行和跨月继承是体系核心，不是次要功能
7. 结果视图必须能支撑：
   - 月度收支分析
   - Rolling Table
   - 持仓明细与收益
   - 负债余额汇总
