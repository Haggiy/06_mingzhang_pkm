# 新明账 AGENTS 文档重构设计

> 日期：2026-04-17
> 状态：已确认
> 主题：将 `AGENTS.md` 从通用贡献指南重构为新明账前置研究仓库协作规范

---

## 1. 背景

`06_mingzhang_pkm` 不再是旧 `01_mingzhang` 的延续开发仓库，而是新明账项目的起点仓库。当前阶段的唯一重点，是从个人 Excel 记账实践出发，梳理真实对象、字段、流程和规则，为后续 iOS App 提供可靠领域基础。

旧 `01_mingzhang` 中仍有部分可复用的上层理念，但其功能路线、工程架构和大部分产品设想已经被判定为不再继续继承。

## 2. 核心定位

新的 `AGENTS.md` 需要明确以下定位：

- 本仓库是新明账项目的起点仓库
- 当前阶段是 Excel 记账实践分析与领域梳理阶段
- 最终目标是基于这套真实实践构建 iOS App
- 总原则是“先复刻，再抽象，再优化”

如果某项能力在当前 Excel 实践中没有稳定对应物，则默认不进入当前阶段范围。

## 3. 当前阶段边界

### 当前要做

- 梳理账月流程、字段语义、分类体系、固定行、摊销、结转、补记、汇总规则
- 记录哪些内容已被样例证明，哪些仍是推断，哪些尚待确认
- 形成未来 iOS 设计可直接消费的领域文档与样例说明

### 当前不做

- 不提前设计完整产品 roadmap
- 不提前设计数据库、API、页面结构或技术架构
- 不继承旧项目的复杂功能方案，如预算、预测、复杂看板、Agent 能力

## 4. 与 01_mingzhang 的关系

### 可继承

- `要理财，先明账`
- 以资产、负债、收入、支出理解个人财务
- 用户操作贴近真实记账习惯，系统内部再整理更严格的财务语义

### 不继承

- `J0-J12` 功能分期
- 旧 Web 应用架构、API、数据库和权限方案
- 任何没有被当前 Excel 实践验证的具体功能设想

判断规则：

- 若旧项目中的一个想法，无法回答“它在当前 Excel 实践里对应什么”，则只可作为后续启发，不能写入当前阶段规范。

## 5. 协作规则

- Excel / CSV / XLSX 样例文件是第一证据
- 用户口述与当前仓库已确认文档是第二层证据
- `01_mingzhang` 只能提供理念启发，不能压过当前事实
- 所有结论必须标注为 `事实`、`推断` 或 `待确认`
- 当前阶段的有效推进必须体现为：澄清一个真实规则、界定一个边界、沉淀一份未来 iOS 可消费的领域文档

## 6. 文档结构决策

新的 `AGENTS.md` 保留标题 `Repository Guidelines`，正文使用中文，结构包含：

1. 仓库定位
2. 当前阶段范围
3. 与 `01_mingzhang` 的关系
4. 证据优先级与结论标注
5. 有效产出与协作方式

文档不再强调构建、测试、PR 流程，而是强调研究阶段的方向约束和协作边界。
