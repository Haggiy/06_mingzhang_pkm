# 新明账 iOS MVP PRD

> 状态：权威 PRD 入口
> 来源：`2026-04-27-mvp-product-skeleton-design.md` 讨论确认内容整理
> 范围：复刻当前 Excel 复式记账工作流的 iOS MVP

## 1. 产品定位

`事实`

- 当前 Excel 实践以 `流水日记账` 为唯一最终入账来源。
- 当前已经确认资产负债、结果分析、投资明细等能力都必须回到 `流水日记账` 形成正式记录。
- 当前不维护账月状态、结账状态、核对完成状态。

`推断`

新明账 iOS MVP 的目标是：

```text
复刻当前 Excel 复式记账工作流
-> 用 App 入口降低操作成本
-> 用实时闭环引擎维持资产、负债、收入、支出的语义一致
```

MVP 不是通用财务软件，也不是预算、预测或自动理财产品。当前阶段只做已经被 Excel 实践验证过的能力。

## 2. 一级入口

`事实`

用户确认 MVP 采用五个一级入口：

1. `首页`
2. `流水`
3. `资产负债`
4. `结果视图`
5. `设置`

`推断`

五个入口的职责如下：

| 入口 | MVP 职责 |
| --- | --- |
| `首页` | 展示当前账月摘要、最近记录和快速入口 |
| `流水` | 承接新增、导入、编辑、继承和所有正式入账记录 |
| `资产负债` | 展示从流水推导出的资产负债对象，并承接基金投资明细 |
| `结果视图` | 展示月度结果、收支结构、资产负债变化和投资结果 |
| `设置` | 维护记账语义配置、数据管理和 App 通用设置 |

核心约束：

```text
只有流水日记账中的记录才算正式入账。
首页、资产负债、结果视图都不是独立真源。
设置不承接日常记账主流程。
```

## 3. MVP 数据流

`推断`

MVP 的最小数据流为：

```text
手工记录 / 支付宝导入 / 微信导入 / 投资明细记录
-> 流水日记账记录
-> 实时存量-流量闭环引擎
-> 必要骨架行写回流水日记账
-> 资产负债对象视图
-> 结果视图
```

关键规则：

- `手工记录` 用于账单外交易、补记、调整、冲销。
- `支付宝导入` 和 `微信导入` 是 MVP 必做导入来源。
- `投资明细记录` 先在基金投资明细中计算，再回填到流水日记账。
- 引擎计算结果只有写成流水日记账记录后，才进入正式结果。
- 银行/信用卡导入及其他来源导入不进入 MVP。

## 4. MVP 必做范围

`推断`

### 4.1 首页

- 当前账月收入、支出、结余摘要。
- 当前账月收支结构摘要。
- 最近记录。
- 快速记账入口。
- 查找入口。
- 进入配置和其他页面的入口。

### 4.2 流水

- 账月列表。
- 账月工作区。
- 手工新增、编辑、删除记录。
- 支付宝账单导入和整理。
- 微信账单导入和整理。
- 导入后逐条或批量修改账月。
- 时间字段支持到秒。
- 金额采用当前科目的带符号发生额语义。
- 记录来源识别。
- 继承行。
- 继承用途：复式骨架、周期重复。
- 引擎骨架行写回流水日记账。
- 投资明细回填流水日记账。

### 4.3 资产负债

- 资产总览。
- 负债总览。
- 基金投资资产。
- 基金投资明细账。
- 从资产负债对象回溯来源流水。
- 从负债总览发起还款记录。
- 从负债总览发起利息补记。

### 4.4 结果视图

- 月度结果总览。
- 收支结构。
- 资产负债变化。
- 投资结果。
- 从结果回溯来源流水或基金投资明细。

### 4.5 设置

- 收付手段维护。
- 收付手段属性：资产型、负债型。
- 收付类型维护。
- 类型明细维护。
- 语义标签配置。
- 数据导出、备份、恢复。
- 外观、隐私、安全、关于。

## 5. MVP 明确不做

`推断`

第一版明确排除以下能力：

- 账月状态。
- 结账或锁账。
- 待处理任务中心。
- 以次月账单补记作为负债首次入账条件。
- 会计等式视图。
- 通用借贷分录。
- 多现金账户独立维护。
- 银行/信用卡导入及其他来源导入。
- 完整信用卡账单管理。
- 导入字段映射用户配置。
- 账月规则配置。
- 继承全局偏好配置。
- 自动识别所有递延对象。
- 非基金投资品类。
- 复杂投资组合分析。
- 实时行情。
- 预算。
- 预测。
- AI / Agent。
- 自动理财建议。
- 高级图表看板。

## 6. 关键验收场景

`推断`

MVP 至少要跑通以下 12 条场景：

1. 用户在首页查看当前账月收支分析和最近记录，并通过快速入口记账或查找。
2. 用户手工新增一条记录，时间精确到秒，保存后进入当前账月流水。
3. 用户导入并整理一批支付宝流水，系统按实际时间预填账月，用户可以逐条或批量调整账月归属。
4. 用户导入并整理一批微信流水，系统按实际时间预填账月，用户可以逐条或批量调整账月归属。
5. 用户在 2026-03 记录一笔信用卡餐饮消费，3 月结果体现支出增加，广发卡负债同步增加，现金资产不变。
6. 用户在 2026-04 记录信用卡还款，现金资产减少，负债减少。
7. 用户查看负债总览，可以看到信用支付形成的负债、利息成本、已还款和剩余负债。
8. 用户新增一条基金投资卖出记录，投资明细账按平均成本法算出入账金额，并回填流水日记账。
9. 用户记录一笔预付费用，后续账月通过继承行释放递延资产并确认消费。
10. 用户在结果视图发现某个结果异常，可以追溯到流水来源行。
11. 用户在设置中维护收付手段、收付类型、类型明细和语义标签。
12. 用户可以导出、备份、恢复数据。

## 7. v1.0 已确认边界口径

`事实`

2026-05-09，用户确认 v1.0 先按以下口径实现：

- 投资资产：先做 `总投资资产`。
- 递延资产对象：先用 `备注` 识别。
- 现金池：先合并。
- 负债对象：先按收付手段聚合。

`推断`

以上口径只冻结 v1.0，不否定后续版本按投资账户、显式递延对象、具体现金账户或账单周期继续细化。
