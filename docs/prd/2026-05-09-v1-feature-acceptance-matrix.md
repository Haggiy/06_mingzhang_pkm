## 1. v1.0 范围原则

`事实`

- v1.0 采用五个一级入口：`首页`、`流水`、`资产负债`、`结果视图`、`设置`。
- `流水日记账` 是唯一最终入账和计算真源。
- 首页、资产负债、结果视图都不是独立真源。
- 设置不承接日常记账主流程。
- 支付宝导入和微信导入是 MVP 必做导入来源。
- 当前不维护账月状态、结账状态、锁账状态。
- 第一版投资能力只做基金。

`推断`

v1.0 的产品边界应按以下原则冻结：

- 能从当前 Excel 实践、PRD 或 Spec 找到对应物的能力，才进入 v1.0。
- 所有用户可见结果必须能追溯到流水记录或基金投资明细。
- 手工记录、导入确认记录、投资明细回填记录是真源记录；engine 记录参与计算但不触发引擎。
- 第一轮开发按纵向闭环推进，不按页面横向铺满。
- v1.0 先按纯本地 App 口径理解，不默认引入账号、多端同步或云服务。

## 2. 功能清单

### 2.1 P0：最小纵向闭环

`推断`

P0 功能用于跑通第一条真实价值闭环：

```text
手工新增信用卡餐饮消费
-> 保存为流水真源记录
-> 引擎解释为支出 + 负债
-> 首页、资产负债、结果视图同步变化
-> 可以回溯到来源流水
```

| 功能编号 | 功能 | 范围说明 | 完成标志 | 依据 |
| --- | --- | --- | --- | --- |
| `F-P0-01` | 本地账本基础 | 支持保存和读取账月、流水记录、基础配置和引擎记录 | 可以新增、读取、修改、删除一条流水记录 | `PRD`、`MONTH`、`ROADMAP` |
| `F-P0-02` | 账月工作区 | 记录同时具备 `账月` 与 `时间`；账月是归属容器，时间是事实发生时间 | 当前账月可查看；历史账月记录可被修改并触发必要重算 | `MONTH` |
| `F-P0-03` | 手工新增记录 | 支持账月、时间、收付手段、金额、收付类型、类型明细、备注 | 新增记录以 `record_source = manual` 进入对应账月流水 | `PRD`、`MONTH` |
| `F-P0-04` | 流水列表与详情 | 按账月查看流水，支持详情、编辑、删除和来源标记 | 用户能从列表进入详情并修改真源记录 | `PRD`、`MONTH` |
| `F-P0-05` | 存量-流量引擎最小规则 | 支持信用支付形成负债、支出分类汇总、当前账月重算、engine 记录覆盖 | 信用卡消费能让支出增加、负债增加、现金不变 | `ENGINE`、`ROADMAP` |
| `F-P0-06` | 首页摘要 | 展示当前账月收入、支出、结余、最近记录和快速入口 | 摘要来自真实流水和引擎结果，不使用写死数据 | `PRD`、`ROADMAP` |
| `F-P0-07` | 资产负债总览 | 展示资产、负债及对象摘要，至少支持负债型收付手段聚合 | 信用卡消费后对应负债对象余额增加 | `PRD`、`ENGINE` |
| `F-P0-08` | 结果视图总览 | 展示月度结果、收支结构和支出分类摘要 | 生活必要开支等分类结果可由流水推导 | `PRD`、`ENGINE` |
| `F-P0-09` | 最小查找与回溯入口 | 首页查找入口可进入流水查询；首页、资产负债、结果视图中的结果能回到来源流水 | 从首页能进入流水查找；从异常结果能定位到相关流水记录 | `PRD` |

`推断`

| --- | --- | --- | --- | --- |

| `F-P1-04` | 收付手段维护 | 支持新增、编辑、停用，并标记资产型、负债型、账务处理型 | 新旧流水能保留历史收付手段语义 | `SETTINGS` |

| `F-P1-07` | 基金投资明细账 | 记录基金交易事实，展示计算字段 | 用户维护交易事实，系统计算平均成本和投资结果 | `PRD`、`INVEST` |
| `F-P1-08` | 投资回填流水 | 买入、卖出、收益、亏损按账月回填流水 | 回填记录使用 `record_source = investment_feed`，结果可追溯到投资明细 | `INVEST`、`ENGINE` |
| `F-P1-09` | 递延资产与继承行 | 支持预付费用后续账月释放，继承行承接复式骨架或周期重复 | 后续账月能通过继承行释放递延资产并确认消费 | `MONTH`、`ENGINE` |
| `F-P1-10` | 数据导出、备份、恢复 | 支持核心账本导出、备份文件生成、恢复 | 重装或换设备场景下可恢复核心账本 | `PRD`、`SETTINGS`、`ROADMAP` |

`推断`

P2 不改变 v1.0 核心能力，只在 P0/P1 稳定后补齐体验。

| 功能编号 | 功能 | 范围说明 | 完成标志 | 依据 |
| --- | --- | --- | --- | --- |

| `F-P2-03` | 空状态与错误状态 | 各入口补齐无数据、导入失败、恢复失败等状态 | 用户能理解当前状态和下一步操作 | `ROADMAP` |

## 3. v1.0 明确不做

`事实`

以下能力已在 `PRD` 或 Spec 中排除，不进入 v1.0 开发范围：

- 账月状态、结账、锁账。
- 待处理任务中心。
- 以次月账单补记作为负债首次入账条件。
- 通用借贷分录。
- 会计等式视图。
- 多现金账户独立维护。
- 银行和信用卡账单导入。
- 用户自定义导入字段映射。
- 复杂自动分类规则。
- 账月规则配置。
- 继承全局偏好配置。
- 非基金投资品类。
- 实时行情。
- 预算、预测、AI / Agent、自动理财建议。
- 高级图表看板。
- 账号系统、多设备云同步、团队协作账本。

`推断`

如果后续开发中出现上述需求，只能作为后续版本候选，不应混入 v1.0 任务拆分。

### A-08：基金卖出与平均成本法回填

`事实`

第一版投资能力只做基金；卖出入账金额使用平均成本法，不能用交易金额替代。

`推断`

| 项目 | 内容 |
| --- | --- |
| 对应 PRD 场景 | 8. 用户新增一条基金投资卖出记录，投资明细账按平均成本法算出入账金额，并回填流水日记账 |
| 覆盖功能 | `F-P1-07`、`F-P1-08` |
| 前置数据 | 某基金已有历史买入和持有成本；存在可卖出份额 |

## 5. 第一轮开发验收样例

`推断`

第一轮开发只验收 P0 最小纵向切片：

```text
手工新增信用卡餐饮消费
-> 流水入账
-> 引擎形成支出 + 负债
-> 首页、资产负债、结果视图变化
-> 回溯到流水
-> 修改 / 删除后结果重算
```

验收数据：

| 字段 | 值 |
| --- | --- |
| 账月 | `2026-04` |
| 时间 | `2026-04-15 12:30:00` |
| 收付手段 | `广发卡` |
| 金额 | `100` |
| 收付类型 | `生活必要开支` |
| 类型明细 | `伙食费` |
| 备注 | `午餐` |
| record_source | `manual` |

预期：

- 流水出现该记录。
- 结果视图 / 统计中 `生活必要开支` 增加 `100`。
- 资产负债中 `广发卡` 负债增加 `100`。
- 首页支出增加 `100`。
- 现金类资产不减少。
- 修改金额为 `120` 后，上述结果同步变为 `120`。
- 删除记录后，上述影响消失。
