# 设置与记账语义 Spec

> 状态：权威 Spec  
> 范围：收付手段、收付类型、类型明细、语义标签和 MVP 设置边界

## 1. 设置定位

`事实`

- 用户确认设置需要区分 App 普遍设置和明账记账设置。
- 用户确认导入字段映射不做用户配置。
- 用户确认账月归属在记录上修改，不做复杂账月规则配置。
- 用户确认继承偏好不先做全局设置，继承在流水行上处理。

`推断`

MVP 设置分为：

1. `记账配置`
2. `数据管理`
3. `App 设置`

设置用于维护长期配置，不承接日常记账主流程。

## 2. 收付手段

`事实`

当前实践中的收付手段包括三类：

- 资产型：`现金`、`电子钱包余额`
- 负债型：`花呗`、`广发卡`、`招商卡`、`平安卡`、`北京卡`、`蚂蚁卡`、`京东白条`
- 账务处理型：`账务处理`

`推断`

MVP 支持维护收付手段：

- 新增。
- 编辑。
- 停用。
- 标记属性。

最小属性：

```text
资产型 / 负债型 / 账务处理型
```

语义：

- `资产型` 表示现金或类现金资产来源。
- `负债型` 表示信用支付、贷款、白条等负债来源。
- `账务处理型` 表示没有真实资金流动，但需要进行确认、结转、摊销、归集或汇总。

## 3. 收付类型与类型明细

`事实`

当前实践采用三级科目语义：

```text
会计要素 -> 收付类型 -> 类型明细
```

当前主表中的用户字段是：

- `收付类型`
- `类型明细`

`类型明细` 规则：

- 必须唯一归属于一个 `收付类型`。
- 不是自由标签。
- 不跨 `收付类型` 复用。

`推断`

MVP 默认沿用当前 Excel 体系。

收付类型包括：

- 消费类：`生活必要开支`、`文娱游购开支`、`财务费用开支`、`投资自身开支`、`其他必要开支`
- 收入类：`工作收入`、`理财收入`、`其他收入`
- 资产负债类：`资产类支出`、`负债类增记`、`负债类减记`

设置中支持维护：

- 收付类型。
- 类型明细。
- 类型明细所属收付类型。
- 停用不再使用的类型明细。

## 4. 语义标签

`推断`

语义标签用于让系统理解字段背后的财务含义，不替代用户可见分类。

最小用途：

- 让引擎识别资产、负债、收入、支出方向。
- 支撑实时闭环计算。
- 支撑资产负债对象视图。
- 为后续自动化或 AI 能力保留结构化基础。

可标记对象：

- 收付手段。
- 收付类型。
- 类型明细。

示例：

| 对象 | 用户可见值 | 语义标签示例 |
| --- | --- | --- |
| 收付手段 | 广发卡 | `负债型` |
| 收付手段 | 电子钱包余额 | `资产型` |
| 收付类型 | 资产类支出 | `资产` |
| 类型明细 | 长期待摊费用 | `递延资产` |
| 类型明细 | 投资亏损 | `已实现投资亏损` |

## 5. 权益语义

`事实`

当前 Excel 实践没有稳定的权益类用户分类。

`推断`

MVP 中：

- `资产 / 负债 / 收入 / 支出` 作为内部语义保留。
- `权益` 不作为用户可见分类加入。
- 未来如需处理期初净资产、历史调整、权益类概念，可以作为内部扩展。

## 6. 数据管理

`推断`

MVP 数据管理保留：

- 数据导出。
- 数据备份。
- 数据恢复。

暂不做：

- 复杂多设备同步。
- 数据版本管理。
- 自动冲突合并。
- 清空数据快捷入口。

如果未来提供清空数据，必须作为高风险操作，并要求明确确认。

## 7. App 设置

`推断`

App 通用设置可以包含：

- 外观。
- 隐私。
- 安全。
- 关于。

这些设置不影响流水入账语义。

## 8. MVP 不进入设置

`推断`

以下能力不进入 MVP 设置：

- 导入字段映射配置。
- 账月规则配置。
- 继承全局偏好配置。
- 复杂自动化规则中心。
- 日常新增账目。
- 资产负债余额编辑。
- 结果修正。
